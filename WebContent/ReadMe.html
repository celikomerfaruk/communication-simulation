Read Me (v20181110)

Modification

  1. All your modifications must be between the markers:
    //DO_NOT_EDIT_ANYTHING_ABOVE_THIS_LINE
    //DO_NOT_EDIT_ANYTHING_BELOW_THIS_LINE
    All changes in other places will be lost.
  2. If you modify/delete the markers, we will not be able to extract your answer, and you may receive zero. SO, DON'T TOUCH THE MARKERS.

Testing during exam

You are given a test suit to check your code in codes.teaching.testing.student package. We recommend that you use these during exam. It will give a sense of what is expected.
  • Run codes.teaching.testing.student.ExamJUnitCases as JUnit.
  • Note that these test cases do not cover all possible cases to be used in grading.
  • These test cases are not used in grading. So passing these test does not guarantee to get full score in the exam.
  • There may be additional test cases that we expect you to think about.

Submission

  • We will grade only your last submission before the deadline. All submissions after the deadline will be simply ignored.
  • Make sure your code compiles before you submit.
  • It is recommended that you submit your code every time you pass a new test case.

Good Luck!
